Description

기후, 환경, 질병의 연관 통계를 위한 데이터 플랫폼 구현 프로젝트.
사용자 페이지와 관리자 페이지가 구분되어 있음. 사용자는 업로드된 데이터를 1-2개 선택하여 지역/기간별로 통계 분석을 할 수 있고, 결과에 대한 ai의 분석 수행. 원하는 데이터가 없으면 데이터 요청 가능. 관리자는 데이터/사용자 통계 확인 및 관리함.

Reflection

초반에 문서화가 필요한 작업을 제대로 해놓지 않고 개발을 시작하면서 요구사항 변경이 반복되어 혼란을 겪었습니다. 특히 GPT API를 프론트에서 호출했다가 보안 문제를 인지하고 백엔드로 급히 이전한 경험을 통해 설계 초기 단계에서 보안과 역할 분리를 고려하는 것의 중요성을 깨달았습니다.
또 초기 CI/CD 파이프라인 구축 시에는 Jenkins를 사용했는데, 이를 위해 서버를 하나 더 관리하는 것이 프로젝트 규모에 비해 과하다고 느껴졌고, 매번 대시보드로 들어가서 확인을 하는 것이 번거로웠습니다. 깃허브에서 PR 연동을 하려면 GitHub Pull Request Builder 플러그인을 사용해야 했는데 보안 경고가 떠서 안전성을 고려해 Actions 기반으로 전환 했습니다. 덕분에 PR 생성과 동시에 자동 빌드·배포가 이루어지고, GitHub 상에서 바로 결과를 확인할 수 있어 편해졌습니다. 이로써 기술스택 선정에는 기술에 대한 충분한 이해가 필요하다는 것을 몸소 느꼈습니다.
프로젝트 막판에는 백엔드 코드를 전체적으로 리팩토링하여 프로젝트 구조와 응답을 통일시켜서 프론트와 간단히 통신할 수 있도록 했습니다. 프로젝트 설계의 미숙함에도 불구하고 1주일 단위 스프린트 기반의 애자일 개발 방식을 통한 논의와 조율의 시간, 팀원 간의 꾸준한 소통 덕분에 모든 주요 기능을 안정적으로 개발할 수 있었습니다.
