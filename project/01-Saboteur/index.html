Description

실시간 멀티플레이 보드게임 사보타지(Saboteur)의 온라인 구현 프로젝트.
플레이어는 게임 방을 생성하거나 참여할 수 있으며, 방장은 게임 시작 권한을 가짐. 게임은 턴제 방식으로 진행되며, 카드 선택과 행동이 실시간으로 반영됨.
웹소켓 기반 통신과 Redis를 활용한 세션/방 상태 관리로 실시간 동기화를 구현함. 테스트를 통해 다수 인원이 동시에 접속해도 게임이 안정적으로 동작함을 확인함.

Role & Contribution

  • 회원가입 및 로그인 기능 구현: JWT 기반 인증 시스템 설계 및 구현
  • 게임 기능 구현: WebSocket을 활용한 카드 버리기, 카드 가져오기 기능 개발
  • 소프트웨어공학 기반 프로젝트 관리 문서 작성: Project Charter, SRS, API 명세서, UML 다이어그램 등을 통한 체계적 산출물 작성 및 정리 기여

Reflection

이전까지는 프로젝트 틀만 정해지면 곧바로 구현에 들어가곤 했는데, 이번에는 개발 전에 Charter, SRS, API 명세서 등을 먼저 작성하면서 전체 흐름을 먼저 정리했습니다.
처음에는 문서화에 시간을 많이 들이는 것이 분석 마비로 이어지지 않을까 걱정했지만 오히려 큰 혼란 없이 체계적으로 개발을 이어갈 수 있었습니다. 또, SRS 기반 시나리오 테스트를 통해 유닛 테스트로는 놓치기 쉬운 예외 상황들을 미리 잡을 수 있어서 기능 누락 없이 마무리할 수 있었습니다.
WebSocket과 Redis 기반의 실시간 상태 동기화 구현은 처음 경험하는 기술이라 어려웠는데, 요청 흐름을 UML 다이어그램으로 정리하고 명세서를 바탕으로 프론트와의 인터페이스를 명확히 했기에 안정적인 실시간 상호작용을 성공적으로 구현할 수 있었습니다.
문서화를 철저히 하였음에도 각 기능의 난이도에 대한 예측을 잘못하여 작업 분배에 아쉬움이 남아, 더 다양한 경험을 통한 배움에 대한 욕심을 가지게 되었습니다. 그렇지만 문서화 내용에 맞췄기에 팀 전체가 개발한 것이 일관되고, 팀원끼리 코드를 이해하기 쉬워진 것은 저에게 보람과 즐거움을 느끼게 했습니다.
